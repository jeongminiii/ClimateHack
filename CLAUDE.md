> "아이디어 → AI 코드 생성 → 작동 서비스"를 하루 만에 완성하는 바이브코딩 해커톤

## 📊 활용 데이터 (600개+ 데이터셋)

### 5가지 핵심 카테고리

| 분야 | 주요 데이터 | 활용 예시 |
|------|-----------|----------|
| 🌊 **기후위기** | 극한호우·산사태 위험도, 폭염 체감온도, 침수흔적지도 | 재난 대비 앱, 위험지역 알림 |
| 🌳 **도시생태** | 비오톱 유형도, 현존식생지도, 습지·생태축 위치 | 생태 탐방 앱, 녹지 분석 |
| 🔋 **탄소공간** | 건축물 배출량, 수목/토양 탄소 저장량, 흡수지도 | 탄소 발자국 계산기 |
| ☀️ **태양광** | 이론적/기술적/시장 잠재량, 규제지역, 설치 현황 | 태양광 적합지 분석 |
| 🏛️ **공간정보** | 공원, 문화재, 국립공원, 녹지지역 | 위치 기반 서비스 |

## 🌍 경기기후 API 사용법

### 빠른 시작

헬퍼 함수가 이미 포함되어 있습니다: `src/lib/ggClimate.ts`

```tsx
import { getParks, getCulturalProperties, getBiotopTypes } from './lib/ggClimate'

// 공원 데이터
const parks = await getParks(100)
const 수원공원 = await getParks(100, "수원")  // 시군구 필터

// 문화재 데이터
const 문화재 = await getCulturalProperties(100)

// 비오톱 (생태) 데이터
const 비오톱 = await getBiotopTypes(100)

// 아무 레이어나 직접 조회
const 침수위험 = await getLayer("flood_risk_map", 100)
```

### 사용 가능한 함수들

| 함수 | 설명 |
|------|------|
| `getParks(개수, 시군구?)` | 공원 (35,288건) |
| `getCulturalProperties(개수)` | 문화재 (1,759건) |
| `getNationalParks(개수)` | 국립공원 |
| `getGreenAreas(개수)` | 녹지지역 |
| `getBiotopTypes(개수)` | 비오톱 유형 |
| `getLayer(레이어명, 개수)` | 아무 레이어 직접 조회 |
| `getLayerList()` | 전체 레이어 목록 (404개) |

### 인기 레이어

```
park                    - 공원
cultural_property       - 문화재
national_park          - 국립공원
flood_risk_map         - 침수위험지도
biotop_type_evl_5grd   - 비오톱 유형
green_area             - 녹지
```

## 💡 아이디어 예시

## ⚡ 핵심 원칙

### 속도가 전부
- 완벽한 코드 NO, 작동하는 코드 YES
- 타입 에러 무시 (`// @ts-ignore` 활용)
- 린트 에러 무시
- 로컬에서 돌면 OK

### 빌드 체크 최소화
- `npm run build` 체크 안 함
- 로컬 개발 서버에서 작동하면 바로 배포
